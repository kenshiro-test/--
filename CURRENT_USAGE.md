# Dream Schedule Planner 現在仕様

## 概要

Dream Schedule Planner は、東京ディズニーランド / 東京ディズニーシーのショー・パレードを軸に、1日の予定を複数パターンで作成・比較できる Web アプリです。

現在は次の3つが大きな柱です。

1. ホーム画面から日付選択、作成済み予定確認、入園後メモ管理
2. ショー選択と複数パターン比較を含むスケジュール作成
3. 管理者モードによるショー・パレードのマスターデータ / 日別スケジュール管理

## 起動方法

ローカルサーバーを起動します。

```bash
python3 server.py
```

ブラウザで次を開きます。

```text
http://localhost:8000/index.html
```

管理者モードを使う場合は次を開きます。

```text
http://localhost:8000/index.html?admin=true
```

管理者モードを有効化するには、サーバー起動時に `ADMIN_PASSWORD` または `ADMIN_PASSWORD_HASH` を設定します。

```bash
ADMIN_PASSWORD='任意の強いパスワード' python3 server.py
```

HTTPS 配信時に管理者 Cookie に `Secure` を付ける場合は、次のように起動します。

```bash
ADMIN_PASSWORD='任意の強いパスワード' ADMIN_COOKIE_SECURE=true python3 server.py
```

## 主要ファイル

| ファイル | 役割 |
| --- | --- |
| `index.html` | 現在のアプリ本体。HTML / CSS / JavaScript を1ファイルに集約しています。 |
| `server.py` | 静的配信、`/api/data`、管理者認証 API、SQLite 保存を担当します。 |
| `dream_shiori.sqlite3` | 現在の保存先データベースです。 |
| `database.json` | 旧保存形式。初回起動時のみ移行元として使われます。 |
| `supabase-config.js` | 公開版で Supabase に接続するための設定です。 |
| `supabase-schema.sql` | Supabase 側に作成するテーブルと RLS ポリシーです。 |
| `PUBLIC_RELEASE_GUIDE.md` | GitHub Pages + Supabase で公開するための手順です。 |
| `FREE_RELEASE_PLAN.md` | 無料公開を前提にしたリリース方針メモです。 |

## 現在の画面フロー

### 1. ホーム画面

- サイトを開くと最初にホーム画面が表示されます。
- カレンダーから日付を選んで新しい予定を作成します。
- 作成済み予定は `すべて` / `作成済み予定` / `過去の予定` タブで切り替えできます。
- 各予定カードでは `開く` / `表示` / `削除` が使えます。
- `表示` を押すと、ホーム画面上で縮小版スケジュールを展開できます。
- 複数パターンがある予定は、縮小版内の小さいタブで切り替えできます。
- 入園後やることメモ 7 項目を確認 / 編集できます。
- 公式リンクを開けます。

### 2. カレンダーから予定作成

- 日付を押すと確認ダイアログが出ます。
- ショー・パレードのスケジュールが未登録の日付は作成できません。
- 登録済み日付だけが予定作成フローに進めます。

### 3. パーク選択

- ランド / シーを選びます。
- 背景はホーム画面と近い淡いトーンに揃えています。

### 4. ショー選択

- 対象パークのショー・パレード一覧が表示されます。
- 一覧は内側スクロールではなく、最後のショーまで自然に見える構成です。
- `全選択` で一括選択できます。
- 一覧の最後に `スケジュールを作成する` ボタンがあります。

### 5. ショー選択モード

- 選んだショーの公演時間を選び、予定に追加 / 解除できます。
- `＋別パターンを作成` で比較用の別パターンを追加できます。
- `次へ` でスケジュール作成モードへ進みます。

### 6. スケジュール作成モード

- 複数パターンを横並びで比較できます。
- 予定カードの間には空き時間が `75分` のように表示されます。
- 空き時間を押すと、予定追加確認の後に予定追加画面を開けます。
- 空き時間から追加した場合、初期ジャンルは `アトラクション` です。
- 予定カードを押すと編集できます。
- 予定カード右上の削除ボタンで削除できます。
- 右上の操作は `共有` / `画像保存` / `完了` です。
- `完了` を押すとホーム画面へ戻ります。

### 7. 予定追加 / 編集画面

- 現在は横長の編集画面です。
- 左側に入力フォーム、右側にそのパターンだけのスケジュールプレビューを表示します。
- 右側プレビューは通常のスケジュール作成画面に近い見た目です。
- 予定カード、空き時間の分数表示もプレビューに反映されます。
- 開始 / 終了時間や名称を入力すると、右側プレビューが即時更新されます。

## ユーザー向け機能

### 作成済み予定

- 予定タイトルは `🏰 5月15日(金)の予定` / `🌊 5月15日(金)の予定` 形式です。
- パークごとに色分けされたカードで表示されます。
- `表示` ボタンでホーム内プレビューを開けます。
- `開く` で編集画面に遷移します。
- `削除` で日付単位の予定を削除します。

### 縮小版プレビュー

- ホーム画面上で開けます。
- 左側の時間軸は表示しません。
- 予定カードと空き時間の分数は表示します。
- 複数パターンがある場合は小さなタブで切り替えます。

### 予定パターン

- 日付ごとに複数パターンを持てます。
- パターン名は `予定①` `予定②` のように自動採番されます。
- パターンを削除すると残りの名前も振り直されます。

### 予定追加 / 編集

- ジャンルは次から選べます。
  - `show`
  - `attraction`
  - `food`
  - `rest`
  - `other`
- 時間入力は4桁形式です。
  - 例: `0930`, `1015`, `1830`
- 保存時に内部では `09:30` 形式へ正規化します。
- 開始時間と終了時間から所要時間を自動計算します。
- 既存予定と重複する時間帯は保存できません。

### 空き時間から追加

- `75分` などの空き時間を押すと追加確認が出ます。
- OK を押すと予定追加画面が開きます。
- デフォルトでは `アトラクション` が選択されます。
- ジャンルは画面内で変更できます。
- 開始 / 終了時間にはその空き時間が自動入力されます。

### 入園後やることメモ

- 7項目の初期テンプレートがあります。
- デフォルト項目は次です。
  - DPA（アトラクション有料パス）
  - スタンバイパス
  - プライオリティパス（アトラクション無料パス）
  - DPA（ショー&パレード有料パス）
  - レストラン当日予約
  - レストラン、モバイルオーダー
  - ショーエントリー抽選
- メモ追加 / メモ削除ができます。
- 編集モードでは項目削除、並び替え、タイトル編集ができます。
- `リセット` で初期7項目へ戻せます。

### 共有 / 保存

- `共有` は画像共有を行います。
- 共有非対応端末では画像保存へフォールバックします。
- `画像保存` は PNG 保存です。
- ファイル名は `dream_plan_YYYY-MM-DD.png` 形式です。

## 管理者モード

### ログイン

- `?admin=true` 付き URL で直接ログイン画面を表示します。
- 管理者認証はサーバー側で実施し、成功時に HttpOnly Cookie を発行します。
- 通常ユーザーはショー・パレードのマスターデータを変更できません。

### 管理できる内容

- ランド / シーの切り替え
- ショー・パレード一覧の確認
- 新規追加
- 基本情報の編集
  - 名称
  - 所要時間
  - 抽選有無
  - 基本公演時間
- 日別スケジュール編集
- 日付範囲 / 複数日付への一括適用
- 基本時間の流し込み
- 今後30日入力補助
- ショー単位の日別スケジュール全消去
- 保存 / ログアウト

### 現在の運用

- 公式サイトのショー・パレード情報を確認し、取れた分だけ管理者側へ格納する想定です。
- 公式サイトへの通信が失敗した項目は「更新なし」ではなく「通信失敗」として自動取り込み履歴に残します。
- 毎月10日 10:00 に月次確認タスクを動かし、更新あり / 更新なし / 通信失敗を確認します。
- 休止日は空配列で保存します。
- 休止は「未登録」ではなく「確認済み」として扱います。

## 保存仕様

### サーバー保存

- 予定、メモ、管理者データは `server.py` 経由で SQLite に保存します。
- 保存先は `dream_shiori.sqlite3` です。
- `DATABASE_FILE` 環境変数で変更できます。

```bash
DATABASE_FILE=/path/to/dream_shiori.sqlite3 python3 server.py
```

### 公開版の保存

- `supabase-config.js` に Supabase の `url` と `anonKey` が設定されている場合、公開版は Supabase に保存します。
- 一般ユーザーの予定とメモは Supabase Auth の匿名ユーザーIDごとに `user_data` へ保存します。
- 管理者ログイン時だけ、ショー・パレードのマスターデータと日別スケジュールを `global_data` へ保存します。
- Supabase 未設定の場合は、これまで通り localStorage / `server.py` にフォールバックします。

### データ分離

| データ | 保存単位 | 編集権限 |
| --- | --- | --- |
| ショー・パレードのマスター / 日別公演時間 | 全体共通 | 管理者のみ |
| ユーザーの予定 | 匿名ユーザーごと | 各ユーザーのみ |
| 入園後やることメモ | 匿名ユーザーごと | 各ユーザーのみ |

- 一般ユーザーは `dream_user_id` Cookie により分離されます。
- Chrome と Safari は別ユーザーとして扱われる可能性があります。
- ただし管理者データはブラウザをまたいで共通です。

### `/api/data` の挙動

- 管理者認証済み: 全体データを保存
- 未認証: 自分の予定とメモだけ保存

## API

| エンドポイント | メソッド | 役割 |
| --- | --- | --- |
| `/api/data` | `GET` | 全体データ + 自分の予定 / メモを合成して返します。 |
| `/api/data` | `POST` | 管理者は全体データ、一般ユーザーは自分の予定 / メモを保存します。 |
| `/api/admin/status` | `GET` | 管理者モードの有効状態と認証状態を返します。 |
| `/api/admin/login` | `POST` | 管理者ログインします。 |
| `/api/admin/logout` | `POST` | 管理者ログアウトします。 |

## 現在の注意点

- `index.html` に処理が集約されているため、機能追加時に見通しが悪くなりやすいです。
- `server.py` を通さず `file:///.../index.html` で直接開くと、ブラウザごとの `localStorage` 依存になり、共通データ運用に向きません。
- Google Fonts と `html2canvas` は外部読み込みです。
- `parkHours` はデータ上はありますが、現行 UI では編集 UI を出していません。

## 改善候補

今の状態で特に追加価値が高そうなのは次です。

1. 予定の自動整列補助
   空き時間に「この時間に入りそうなアトラクション候補」や「食事候補」を出せると、予定作成がかなり早くなります。

2. 予定の複製
   `5月15日の予定を複製して5月22日に流用` のような機能があると、再訪時に楽です。

3. 公開向け注意表示
   ホームの下部に小さく `非公式ツール / 最新情報は公式サイトをご確認ください` を入れると公開時に安心です。

4. 管理者更新の差分表示
   月次更新時に `前回から追加・削除・時間変更` を一覧で確認できると、運用ミスを減らせます。

5. 予定の共有強化
   今は画像共有中心なので、将来的には `閲覧専用URL` を作れると友達と共有しやすいです。

6. ホームのプレビュー精度向上
   縮小版プレビューはすでにありますが、予定数が多い日向けに自動縮尺をもう少し細かく調整するとさらに見やすくなります。
