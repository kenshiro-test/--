# Dream Schedule Planner 現在仕様

## 概要

Dream Schedule Planner は、東京ディズニーランド / 東京ディズニーシーで見るショーやパレードを選び、1日の予定表を作るためのブラウザアプリです。

ユーザーはホーム画面から作成済み予定、入園後メモ、公式リンクを確認できます。新規予定作成では日付とパークを選び、見たいショーを絞り込んだあと、複数の予定パターンを作成して比較できます。完成した予定は画像として保存できます。

## 起動方法

ローカルサーバーを起動します。

```bash
python3 server.py
```

起動後、ブラウザで次を開きます。

```text
http://localhost:8000/index.html
```

管理者モードを使う場合は、URL に `?admin=true` を付けて開きます。

```text
http://localhost:8000/index.html?admin=true
```

管理者モードを有効にするには、サーバー起動時に `ADMIN_PASSWORD` または `ADMIN_PASSWORD_HASH` を設定します。本番では `ADMIN_PASSWORD_HASH` を推奨します。

```bash
ADMIN_PASSWORD='任意の強いパスワード' python3 server.py
```

ハッシュ値は次のコマンドで生成できます。

```bash
python3 make_admin_password_hash.py
```

出力された `ADMIN_PASSWORD_HASH=...` を公開先の環境変数に設定します。

HTTPS 配信環境で管理者 Cookie に `Secure` 属性を付ける場合は、次のように起動します。

```bash
ADMIN_PASSWORD='任意の強いパスワード' ADMIN_COOKIE_SECURE=true python3 server.py
```

## 主要ファイル

| ファイル | 役割 |
| --- | --- |
| `index.html` | 現在のメインアプリ本体。HTML、CSS、JavaScript がこのファイル内にまとまっています。 |
| `server.py` | 静的ファイル配信と `/api/data` の読み書きを行う簡易 Python サーバーです。 |
| `dream_shiori.sqlite3` | サーバー保存用の SQLite データベースです。初回起動時に自動作成されます。 |
| `database.json` | 旧保存形式です。初回起動時に存在する場合のみ SQLite へ初期移行されます。 |
| `make_admin_password_hash.py` | 本番向けの管理者パスワードハッシュを生成します。 |
| `app.js` | 旧実装と思われるメインロジックです。現在の `index.html` からは読み込まれていません。 |
| `data.js` | 旧実装と思われる初期データ / DataManager です。現在の `index.html` からは読み込まれていません。 |
| `style.css` | 旧実装と思われるスタイルです。現在の `index.html` からは読み込まれていません。 |
| `*.jpg`, `header.png` | ショー画像などの静的アセットです。現行 `index.html` の初期データでは `imageUrl` は空です。 |

## 画面フロー

1. ホーム画面
   - サイトにアクセスすると最初にホーム画面が表示されます。
   - 作成済み予定が「5月6日の予定」のように一覧表示されます。
   - 予定はホームから開く、削除することができます。
   - 入園後やることメモを追加、完了、削除できます。
   - 東京ディズニーリゾート公式サイト、ランド、シー、公式アプリへのリンクがあります。
   - 「新規予定作成」から予定作成フローに進みます。

2. 日付選択
   - 初回表示では「いつ行きますか？」画面が表示されます。
   - 日付入力後、「次へ進む」でパーク選択に進みます。

3. パーク選択
   - ランドまたはシーを選びます。
   - 選択後、自動的にショー選択画面へ進みます。

4. 見たいショー選択
   - 対象パークのマスターショー一覧が表示されます。
   - チェックボックスで表示対象のショーを選べます。
   - 「全選択 / 全解除」で一括切り替えできます。

5. ショー選択モード
   - 選んだショーの公演時間が横並びのレーンで表示されます。
   - 公演時間をクリックすると予定に追加 / 解除できます。
   - 他の予定と時間が重なる場合は追加できません。
   - 「＋別パターンを作成」で比較用の予定パターンを追加できます。
   - 「次へ」でスケジュール作成モードに進みます。

6. スケジュール作成モード
   - 予定パターンごとの列が表示されます。
   - 予定の間には空き時間が表示されます。
   - 各パターンに次のカテゴリの予定を追加できます。
     - ショー
     - 乗物
     - 食事
     - 休憩
     - その他
   - 予定カードをクリックすると編集できます。
   - 予定カード内の削除ボタンで予定を削除できます。
   - 「画像保存」で予定表を PNG として保存できます。

## ユーザー向け機能

### ホーム

- 作成済み予定を日付別、パーク別に一覧表示します。
- 予定一覧には予定パターン数と予定件数が表示されます。
- 作成済み予定は「開く」から編集できます。
- 作成済み予定は「削除」から削除できます。
- 入園後やることメモを追加、完了、削除できます。
- 公式リンクから東京ディズニーリゾート関連ページを開けます。

### 予定パターン

- 日付、パークごとに複数の予定パターンを持てます。
- パターン名は `予定①`, `予定②` のように自動で振り直されます。
- スケジュール作成モードでは複数パターンを横並びで比較できます。
- パターン削除後も残ったパターン名は連番に更新されます。

### 予定追加 / 編集

- 予定は名称、開始時間、終了時間を入力して作成します。
- 所要時間は終了時間から自動計算されます。
- 終了時間が開始時間以前の場合は保存できません。
- 既存予定と時間が重なる場合は保存できません。
- カテゴリごとにカード色とアイコンが変わります。

### 画像保存

- `html2canvas` を使ってスケジュール部分を PNG 保存します。
- ファイル名は `dream_plan_YYYY-MM-DD.png` 形式です。
- 保存対象は `.view-frame` 内のグリッドです。

### リセット

- ヘッダーの「リセット」からパターン初期化モーダルを開けます。
- 初期化対象はランド / シーの `plans` です。
- 管理者モードで設定したショーマスタや日別スケジュールは保持されます。

## 管理者モード

`?admin=true` 付きで開くと、通常の予定作成フローは表示せず、最初に管理者ログイン画面を表示します。サーバー側で `ADMIN_PASSWORD` が設定されている場合のみログインできます。

管理者パスワードはフロントエンドには保持せず、`/api/admin/login` に送信してサーバー側で検証します。ログインに成功すると、サーバーが HttpOnly Cookie を発行し、そのセッション中だけ管理者 UI が有効になります。

`ADMIN_PASSWORD` と `ADMIN_PASSWORD_HASH` がどちらも未設定の場合、管理者モードは無効です。通常ユーザー向けの予定作成機能は引き続き利用できます。

### 管理者機能

- サーバー側パスワードによるログイン
- ランド / シーの切り替え
- ショーマスタ一覧の確認
- ショーの新規追加
- ショー基本情報の編集
  - ショー名
  - 所要時間
  - 抽選有無
  - デフォルト公演開始時間
- ショー削除
- 今後30日分の日別スケジュール編集
- 日付範囲または複数日付への一括スケジュール適用
- 今後30日分の日付を一括入力欄へ自動投入
- 基本公演時間を一括入力欄へ反映
- 選択中ショーの日別スケジュール全消去
- 管理者変更の保存
- ログアウト

### 管理者画面の使い方

1. 左側でランド / シーを切り替えます。
2. 左側の一覧から編集するショー・パレードを選びます。
3. 「新規追加」でショーを追加できます。
4. 「基本情報を編集」で名前、所要時間、抽選有無、基本公演時間を編集できます。
5. 「削除」で選択中のショーを削除できます。
6. 一括スケジュール入力では、日付と公演開始時間をまとめて入力できます。
7. 最後に「変更を保存」を押すと SQLite データベースと `localStorage` に反映されます。

### 管理者 API

| エンドポイント | メソッド | 役割 |
| --- | --- | --- |
| `/api/admin/status` | `GET` | 管理者モードが有効か、現在のブラウザが認証済みかを返します。 |
| `/api/admin/login` | `POST` | `ADMIN_PASSWORD` と照合し、成功時に HttpOnly Cookie を発行します。 |
| `/api/admin/logout` | `POST` | 管理者 Cookie を削除します。 |

### 日別スケジュール

管理者モードでは、各ショーごとに日別の公演時間を設定できます。

表示時は次の優先順位で公演時間が使われます。

1. 選択日付の `dailySchedules`
2. マスターショーの `times`

## データ保存仕様

### ブラウザ保存

現行アプリは `localStorage` の `dream_v3` キーを主に使用します。

### サーバー保存

`DataManager.save()` は `localStorage` に保存したあと、`fetch('/api/data', { method: 'POST' })` で SQLite にも保存します。

起動時は `DataManager.syncWithServer()` が `/api/data` を取得し、`parks` 構造を持つデータであれば `localStorage` に反映します。

### SQLite 保存

現行サーバーは `dream_shiori.sqlite3` を使います。保存先は `DATABASE_FILE` 環境変数で変更できます。

```bash
DATABASE_FILE=/path/to/dream_shiori.sqlite3 python3 server.py
```

SQLite には次の2種類のデータを分けて保存します。

| データ | 保存単位 | 編集権限 |
| --- | --- | --- |
| ショー・パレードのマスター、日別公演時間 | アプリ全体 | 管理者のみ |
| ユーザーの予定パターン | 匿名ユーザーIDごと | 各ユーザーのみ |
| 入園後やることメモ | 匿名ユーザーIDごと | 各ユーザーのみ |

一般ユーザーは `dream_user_id` Cookie で匿名識別されます。ログイン機能はありませんが、ブラウザごとに予定が分離されるため、他人の予定とは混ざりません。

### データ構造

```json
{
  "parkHours": {
    "open": "09:00",
    "close": "21:00"
  },
  "parks": {
    "land": {
      "masterEvents": [],
      "plans": {},
      "dailySchedules": {}
    },
    "sea": {
      "masterEvents": [],
      "plans": {},
      "dailySchedules": {}
    }
  }
}
```

### masterEvents

```json
{
  "id": 1,
  "name": "ハーモニー・イン・カラー",
  "times": ["12:45"],
  "isLottery": true,
  "imageUrl": "",
  "duration": 45
}
```

### plans

`plans` は日付ごとの予定パターン配列です。

```json
{
  "2026-05-09": [
    {
      "id": 0,
      "name": "予定①",
      "items": []
    }
  ]
}
```

### items

```json
{
  "masterId": 1,
  "name": "ハーモニー・イン・カラー",
  "time": "12:45",
  "duration": 45,
  "cat": "show"
}
```

カテゴリは現状次の値を使います。

| cat | 表示カテゴリ |
| --- | --- |
| `show` | ショー・パレード |
| `attraction` | アトラクション |
| `food` | 食事 |
| `rest` | 休憩 |
| `other` | その他 |

### dailySchedules

`dailySchedules` は日付ごと、ショー ID ごとの公演時間配列です。

```json
{
  "2026-05-09": {
    "1": ["12:45"],
    "2": ["18:15"]
  }
}
```

## 現在の実装上の注意点

- `index.html` が単独で CSS / JS を内包しており、`app.js`、`data.js`、`style.css` は現行画面から参照されていません。
- `server.py` を使わずに `index.html` を直接開いた場合、`/api/data` への同期は失敗します。ただし `localStorage` 保存は動作します。
- Google Fonts と `html2canvas` は外部 CDN から読み込まれます。ネットワークがない環境ではフォントや画像保存機能に影響する可能性があります。
- 管理者パスワードは `ADMIN_PASSWORD_HASH` または `ADMIN_PASSWORD` 環境変数で設定します。フロントエンド固定値は使いません。
- `/api/data` の POST は通常ユーザーも使いますが、未認証ユーザーの保存は自分の予定パターンだけに制限されます。ショー・パレードのマスターデータと日別公演時間は管理者認証済みの場合だけ保存されます。
- `parkHours` はデータ構造にありますが、現行 UI では開園 / 閉園時間を編集する入力は表示されていません。
- `pickedShowIds` は画面上の一時状態で、永続化されません。再読み込み後はショー絞り込み状態が初期化されます。
